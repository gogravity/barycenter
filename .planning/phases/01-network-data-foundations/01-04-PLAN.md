    Sections:

    ## Topology
    Hub-and-spoke with FortiGate-VM02 as inline NVA. CIDRs: hub 10.10.0.0/22, spoke 10.20.0.0/22.

    ## Recursion guard (CRITICAL)
    PE and data subnets get routeTable: null. Otherwise SQL PE traffic would be forced through
    the FortiGate trust NIC, creating an infinite routing loop on private link traffic.

    ## FortiGate license
    BYOL license is stored in KV secret named "fortigate-license" (placeholder created in
    plan 05). The FortiGate VM's system-assigned identity gets `Key Vault Secrets User`
    role on the KV. Post-deploy CI step:
    ```
    LICENSE_B64=$(az keyvault secret show --vault-name kv-bary-dev --name fortigate-license --query value -o tsv)
    echo "$LICENSE_B64" | base64 -d > /tmp/license.lic
    az vm run-command invoke --resource-group rg-barycenter-dev --name vm-fgt-bary-01 \
      --command-id RunShellScript --scripts "fgt-cli execute restore license tftp:/tmp/license.lic"
    ```
    The plaintext .lic never enters git or GitHub Secrets.

    ## Config-as-code (NETW-01)
    All firewall rules live in `fortigate-config/policies.json`. The plan-08 drift detector
    (`scripts/ci/fortigate_drift.py`) reads this file and diffs it against the live FortiGate
    REST API response. Manual edits via the FortiGate web UI are caught and revoked nightly.

    ## Syslog → Log Analytics (NETW-03)
    `policies.json` contains a `syslog` block with target IP `REPLACED_BY_DEPLOY_PIPELINE`.
    Plan 06's deploy pipeline substitutes this with the LA workspace's ingestion endpoint
