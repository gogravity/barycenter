    Step 3 — Create new secret version in KV:
    ```bash

      --value "$(openssl rand -hex 32)"
    ```
    Record OLD and NEW version IDs from KV.

    Step 4 — Open dual-write window programmatically:
    ```python
    import os
    from azure.identity import DefaultAzureCredential
    from azure.keyvault.secrets import SecretClient
    import pyodbc
    from barycenter.audit import AuditClient
    from barycenter.audit.sinks import LogsAnalyticsSink, WormBlobSink
    from barycenter.etl.framework.salt_rotation import SaltRotation

    cred = DefaultAzureCredential()
    kv = SecretClient(vault_url=os.environ["KEY_VAULT_URL"], credential=cred)
    sql = pyodbc.connect(os.environ["SQL_CONNECTION_STRING"])
    audit = AuditClient(sql, LogsAnalyticsSink.from_env(cred), WormBlobSink.from_env(cred))
    sr = SaltRotation(kv, sql, audit)
    sr.open_window(TEST_TENANT, old_version=OLD, new_version=NEW)
    ```

    Step 5 — Run a dual_write call to populate pseudo.person_map with both versions:
    ```python
    result = sr.dual_write("test@synthetic.example.com", TEST_TENANT,
                           old_version=OLD, new_version=NEW)
    assert result.pid_old != result.pid_new
    ```

    Step 6 — Verify dual-write coverage:
    ```sql
    SELECT salt_version, COUNT(DISTINCT person_pid) as cnt
    FROM pseudo.person_map
    WHERE tenant_id = '<TEST_TENANT>'
    GROUP BY salt_version;
    ```
    Both versions should appear with cnt >= 1.

    Step 7 — Cut over: `sr.cut_over(TEST_TENANT)`. Verify state YAML now shows `mode: new-only`.

    Step 8 — Retire old version (T+30 in production; immediate for fire drill to validate disable path):
